Sequência e volume de pedidos no painel

Cortes PIX vs cartão D+2 e horários

Dois relatos distintos na síntese; frequência diária citada nos dois casos.

Dados necessários

  • Entradas do dia, taxas, estornos, saldo inicial e final, saques/transferências; composição do saldo disponível (o que entrou no período).
  • Em paralelo, quem opera a loja costuma conferir sequência e volume de pedidos no painel antes de cruzar com os recebimentos.

Frequência

  • Diária citada para fechamento de saldo na carteira e conferência do dia (ex.: Wishin, Flor de Cacto).

Ferramentas hoje

  • Nuvemshop / Nuvem Pago: painel de vendas, saldo disponível, lançamentos futuros, exportação de relatório de vendas.
  • Planilhas quando o detalhe ou o vínculo não estão no produto.

Principais desafios

  • Ver quais vendas/parcelas compõem o saldo; histórico de saques/transferências difícil para bater com banco.
  • Horários e cortes (PIX vs cartão D+2, antecipação, madrugada) atrapalham o “fechamento do dia”.

Onde querem acessar

  • Painel para leitura rápida; CSV / export para conferência e ERP; API desejável para automatizar com Bling/Tiny — referido na síntese como necessidade de integração.

C · Bancária — OFX (Ana BPO)

OFX do banco
→
ERP contas a pagar / receber
→
Identificar duplicidade, furos, baixas

A e C com ritmos diários citados; B é fechamento mensal. Conciliação de taxas vs contrato é linha à parte na síntese.

Dados necessários

  • Meio de pagamento: vendas do dia (número, sequência, meio), entradas, taxas, estornos, saldos, saques.
  • ERP: pedidos importados, parcelas com datas, baixa de parcela liberada; taxa esperada vs extrato.
  • Banco: OFX e contas a pagar/receber no ERP para conciliação bancária.

Frequência

  • Conciliação vendas × ERP: diária em operações exigentes; muitos lojistas não fazem (BPO).
  • Conciliação bancária (OFX): diária (ex.: “ontem”) no relato BPO.
  • Conciliação de taxas: muitas vezes mensal; desejo de ritmo maior.

Ferramentas hoje

  • Nuvem Pago + ERP (Bling, Tiny); importação OFX no banco/ERP; planilhas para multi-canal.

Principais desafios

  • Pedido na Nuvem sem taxas → lançamento manual ou divergência; risco de duplicação na importação.
  • Trabalho manual e várias planilhas quando há várias empresas/canais/ERPs.
  • Taxa contratada vs taxa descontada; medo de alteração sem aviso.

Onde querem acessar

  • Extrato detalhado exportável; relatório de vendas; Excel compatível com rotina contábil; API para encaixar no ERP.

Tarefa

DRE (Demonstração de Resultado do Exercício)

Fluxo — DRE (síntese)

Período regime de competência · mensal (BPO)
→
Reunir descontos, frete, devoluções, tarifas por competência, CMV, comissões
→
Fechar demonstração de resultado / margem

Visão só de caixa não substitui competência para a DRE (síntese). Cenário BPO: ver também encadeamento DRE → DFC → OFX em jornadas-gestao-financeira.html.

Dados necessários

  • Descontos (cupom e desconto na peça), frete (pago pelo cliente vs absorvido), devoluções por competência.
  • Tarifas por competência (não só o que saiu no caixa no mês).
  • CMV/custo, comissões de plataforma quando existirem.

Frequência

  • Mensal para fechamento gerencial / DRE no relato BPO.

Ferramentas hoje

  • ERP + exportações; orientação contábil (competência vs caixa); planilhas para fechar o que o e-commerce não oferece nativamente.

Principais desafios

  • Relatório “só de caixa” não substitui visão de competência para DRE.
  • Falta de taxas e faturamento alinhados à competência na exportação.

Onde querem acessar

  • Export rico (CSV) ou API com campos de competência; integração direta ao ERP é o uso típico.

Tarefa

Monitorar estatísticas do negócio

Fluxo — estatísticas e disputas (síntese)

Consumir dados admin / relatórios / export (ex.: receita por meio)
→
Analisar mix de meios, parcelas, taxas, aprovação, antifraude
→
Disputas chargeback e estorno quando ocorre — status, valores, motivo

A síntese não detalha ordem fixa entre análise gerencial e disputas; fluxo linear é esquemático. Ritmo gerencial variável; chargeback/estorno eventual.

Dados necessários

  • % PIX, cartão, parcelas, bandeiras; comparação de taxas para negociação com adquirentes.
  • Taxa de aprovação, antifraude, eventual whitelist.
  • Chargeback/estorno: status, valor devolvido, taxa devolvida, motivo; lista exportável.

Frequência

  • Análise gerencial: ritmo variável; chargeback/estorno quando ocorre.

Ferramentas hoje

  • Estatísticas no admin (ex. receita por meio de pagamento); relatórios exportáveis; planilhas internas / Tableau em alguns casos.

Principais desafios

  • Perda de visibilidade em chargeback após mudança de fluxo (e-mail/forms vs painel).
  • Cruzar estatísticas com decisões comerciais sem export estável.

Perguntas que querem responder (exemplos das entrevistas)

  • Como está o mix de meios e parcelas? A taxa efetiva bate com o contratado?
  • Há queda de aprovação ou bloqueios de antifraude que expliquem queda de conversão?
  • Onde estão disputas e estornos e qual o impacto em caixa?

Parte 6 · Legenda

Matriz dado × tarefa × formato

1. Dado — O que o usuário quer consumir (ex.: total de taxas, parcela 2 de 6).

2. Tarefa — Para que trabalho serve (fechamento de caixa, conciliação, DRE, estatísticas). Um dado pode servir várias tarefas.

3. Formato — Forma ideal levando em conta o destino (ex.: CSV para ERP, API para integração, painel para leitura rápida).

4. Temos o dado hoje? — Sim / Parcial / Não / A validar (conforme síntese e evolução do produto).

Onde o produto costuma expor dados hoje

  • Admin / Extrato Nuvem Pago
  • Relatório de vendas — referência (Drive)
  • Extrato exportável — template Nuvem Pago (“Extrato exportável”)
  • Estatísticas — exemplo admin stats (loja conceito)
